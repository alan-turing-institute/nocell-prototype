4.1 Overview

Nocell is a language for building probabilistic spreadsheets.

Nocell -> Cell -> Sheet -> .ods, .xlsx, etc.

Interacting with nocell inside Racket

Idea: start in the middle (cell), and define a stack/assignment/cell data structure. Then nocell evaluation model. Type conventions, then nocell.

Later, sheet, and converting from cell.

4.2 The nocell language

Each nocell program begins with the line

procedure

(stack-print expr)

Prints the result of evaluating expr in "stack" form. Most useful when used interactively.

procedure

(len a) → (Array number?)

  a : (Array any/c)
Returns the shape of a, as an array of rank one. TODO example

procedure

(nth n a) → number?

  n : (Array number?)
  a : (Array number?)
Return the element of array a with index n, where n is a vector with length equal to the rank of a. Note: at the moment, it is only possible to select from arrays of numbers.

procedure

(if test then else) → any/c

  test : boolean?
  then : any/c
  else : any/c
Conditional. Note that if is a function, and as such it evaluates its arguments. Recursion is permitted, since the call depth is bounded (TODO link to relevant section).

procedure

(halt) → halt?

Returns a special error-value to represent the termination of a calculation.

syntax

(@ expr string ...)

The strings are joined with newlines, and the resulting string is added to the note [TODO note is a link to assignment] of the value resulting from evaluating the expression expr

syntax

(define id expr)

Associates the name id with the result of evaluating expr. If the value of expr does not currently have a name, id is associated with it, without introducing another assignment. On the other hand, if the value of expr already has a name (for example, being the result of a named definition already), then a new assignment is introduced, being a reference to expr and named id
